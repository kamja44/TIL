# 정보보호의 3요소(CIA Triad)

- 기밀성, 무결성, 가용성

# 기밀성(Confidentiality)

- 권한 있는 자만이 정보자산에 접근하여 그 내용을 알 수 있도록 보장한다.
- 정당하지 않은 자에 의해 정보가 노출되지 않도록 보장한다.

# 무결성(Integrity)

- 권한 있는 자만이 정보자산에 접근하여 그 내용을 생성, 변경하거나 삭제할 수 있도록 보장한다.
- 정당하지 않은 자에 의해 정보가 위변조되지 않도록 보장한다.

# 가용성(Availability)

- 권한 있는자의 정보자산 접근 시 언제든지 접근할 수 있도록 보장한다.
- 정당한 자의 정보자산 접근이 언제든지 가능하도록 보장한다.

# 인증(인증성)[Authenticity(Authentication)]

- 인증(인증성)은 사용자 인증과 메시지 인증으로 구분할 수 있다.

## 사용자 인증

- 정보자산에 접근하는 자의 신원이 주장된 실체와 일치함을 보장한다.
- 즉, 사용자의 신원이 올바르다는 것을 보장한다.

## 메시지 인증

- 수신한 메시지가 올바른 상대방이 보낸 메시지임을 보장한다.

# 책임성(Accountability)

- 정보자산에 접근하는 자가 접근 결과에 대해 책임지도록 보장하는 것으로, 책임 추적성과 부인방지로 구분할 수 있다.

## 책임 추적성

- 주체를 식별해서 그 행위를 추적할 수 있도록 보장한다.
- 이를 위해 주체가 유일하게 식별되어야 하고, 주체의 행위를 감시하고 로깅하는 것이 필요하다.

## 부인방지

- 주체가 한 행위를 나중에 부인하지 못하도록 보장한다.

# Diffie-Hellman 키교환 알고리즘

- `공개키 암호방식의 개념을 이용하여 두 사용자 간에 공통의 암호화키를 안전하게 공유`할 수 있는 방법이다.
- 키교환이라는 이름이 붙어있지만, 실제로는 키를 교환하는 것이 아닌 공유할 키를 계산하여 만들어 내는 것이다.
  - 키교환이 아닌 키합의 알고리즘이라고도 불린다.
- 유한체상의 이산대수문제를 풀기 어렵다는 것이 Diffie-Hellman 키교환의 이론적인 기반이 되고 있다.
- Diffie-Hellman 프로토콜 방법에서는 양쪽 통신주체가 KDC 없이 대칭 세션키를 생성한다.
  - 대칭키를 만들기 전, 양쪽 두 개의 p와 g를 선택해야 한다.
  - 여기서 p는 매우 큰 소수로서 300자리가 넘는 십진수(1024비트)이다.
- 이산대수공격과 중간자 공격에 취약하다.

# 접근통제(Access Control) 정책 유형

## 임의적 접근통제(DAC : Discretionary Access Control)

- `사용자의 신원 정보(ID : Identity)와 자원에 대한 사용자의 접근권한을 기반으로, 접근통제를 수행하는 정책이다.`
- 자원의 소유자가 해당 자원에 대한 다른 사용자의 접근권한을 임의로 지정할 수 있으므로, '임의적'이라는 용어를 사용한다.
- 다양한 운영체제의 파일시스템과 DBMS등에 적용되어 사용되고 있다.
- 사용자와 자원 각각에 대해. 세분화된 접근통제가 가능하다.
- 중앙 집중적인 접근통제가 어렵다.
  - 즉, 일관성 있는 접근통제 정책의 적용이 어렵다.
- 사용자의 신원 정보를 기반으로 접근통제를 수행하므로, 신원 정보를 위장한 불법적인 접근이 발생하면 심각한 문제가 발생할 수 있다.
- 트로이목마 공격에 취약하다.

## 강제적 접근통제(MAC : Mandatory Access Control)

- `사용자에게 부여된 보안 취급인가(security clearance)와 자원에 부여된 보안 수준(security level, 보안등급)을 비교`하여, 접근통제를 수행하는 정책이다.
  - 주체와 객체에 부여된 `보안 수준(보안등급)을 비교하여 접근통제를 수행`한다.
  - 주체와 객체에 부여된 `보안 취급허가(취급인가)를 비교하여 접근통제를 수행`한다.
  - 주체와 객체에 부여된 `보안 레이블(security label)을 비교하여 접근통제를 수행한다.`
- 사용자가 임의로 다른 사용자에게 자원에 대한 접근권한을 설정할 수 없고 `관리자(중앙 관리자, 보안 관리자)에 의해서만 `사용자의 보안(비밀) 취급인가와 자원의 보안 수준이 결정되어 접근통제를 수행하므로, '강제적'이라는 용어를 사용한다.
- `군 시스템, 보안 운영체제 등 엄격한 접근통제가 요구되는 환경에 적용`한다.
- MAC 기반의 접근통제 모델로 BLP 모델, Biba 모델, Clark-Wilson 모델, Chinese Wall 모델이 있다.

### BLP(Bell-LaPadula) 모델

- 주체와 객체의 보안 수준(security level)을 비교하여, 접근통제를 수행하는 모델이다.
  - 최초의 수학적 접근통제 모델이다.
- `기밀성(confidentiality)을 보장`하기 위해 정보르 흐름을 통제한다.
- `No Read Up`
  - 사용자보다 보안 수준이 높은 객체의 정보를 읽을 수 없다.
    - 사용자보다 높은 보안 수준의 객체가 가진 정보를 읽음으로써, 정보의 기밀성이 훼손되는 상황을 방지할 수 있다.
- `No Write Down`
  - 사용자보다 보안 수준이 낮은 객체에 정보를 쓸 수 없는 속성
    - 사용자보다 낮은 보안 수준의 객체로 정보가 유출되어 기밀성이 훼손되는 상황을 방지할 수 있다.

### Biba 모델

- 주체와 객체의 무결성 수준(Integrity Level)을 비교하여 접근통제를 수행하는 모델이다.
- `무결성(integrity)을 보장`하기 위해 정보의 흐름을 통제하는 모델 (무결성의 3가지 목표 중 '비인가자에 의한 정보의 위변조 방지'만 해결)
- `No Read Down`
  - 사용자보다 무결성 수준이 낮은 객체의 정보를 읽을 수 없는 속성이다.
    - 주체보다 무결성 수준이 낮은 객체의 정보를 읽음으로써, 주체가 알고 있는 정보의 무결성이 훼손되는 상황을 방지할 수 있다.
- `No Write Up`
  - 사용자보다 무결성 수준이 높은 객체에 정보를 쓸 수 없는 속성이다.
    - 객체보다 무결성 수준이 낮은 사용자의 쓰기에 의해 객체가 가진 정보의 무결성이 훼손되는 상황을 방지할 수 있다.

### Clark-Wilson 모델

- 상업적 응용에서 `무결성의 3가지 목표`를 모두 보장하기 위해 접근통제를 수행한다.

### Chinese Wall 모델

- `이해관계가 충돌하는 상업적 응용에서 기밀성과 무결성을 보장`하기 위해 접근통제를 수행하는 모델이다.

## 역할 기반 접근통제(RBAC : Role Based Access Control)

- `사용자에게 부여된 역할과 역할이 가지는 자원에 대한 접근권한을 기반으로 접근통제를 수행`한다.
  - DAC와 구분하기 위해 '비임의적 접근통제(Non-DAC) 정책'이라고도 한다.
- 역할에 자원에 대한 접근권한을 부여하고, 사용자에게 역할을 할당하는 방식이다.
  - 즉, 사용자가 많고 사용자의 접근권한이 자주 변경되는 환경에서 접근통제 관리가 용이하다.
- `조직 내의 잦은 부서 이동 등의 이유로, 조직이 동적으로 변화하는 구조에 적합`한 정책이다.
- 관리자에 의해 일관선 있는 접근통제 정책 적용이 용이하다.
- 역할을 기반으로 자원에 대한 접근권한을 설정하므로, 직무 분리와 최소 권한 부여가 용이하다.
- 임의적 접근통제 정책에 비해, 사용자와 객체 각각에 대한 접근권한 설정의 유연성이 떨어진다.

# Shodan(쇼단)

- 일반적임 검색엔진과 달리 라우터, 스위치, 특정 웹서버, CCTV 등 다양한 장비와 서비스에 대한 정보를 수집하여 사용자에게 IP 정보, 국가 정보, OS 정보, 포트 정보 등의 결과를 알려주는 검색엔진이다.
  - 인터넷에 연결된 모든 기기가 검색의 대상이다.
- 각 기관(회사)에서 관리하는 장비와 서비스의 보안 취약점을 쉽게 파악할 수 있도록 하기 위해 사용한다.
  - 이를 악의적인 목적으로 활용할 수 있다.

# 인터넷(웹) 공간의 접근 형태

## 표층웹(Surface Web)

- 일반적으로 사용하는 웹 브라우저를 통해 구글, 네이버 등의 검색엔진에 접속하여 검색어를 입력한 후 접근할 수 있는 일반인에게 공개된 웹 사이트 공간이다.

## 딥웹(Deep Web)

- 표층웹과 달리 일반적인 검색엔진 사이트를 통해 검색되지 않는 비공개 인터넷 영역이다.
- 개인 이메일, 회사 내부망, 유로 DB 사이트 등이 해당한다.

## 다크웹(Dark Web)

- 딥웹의 일종으로 특수한 브라우저를 통해서만 접근이 가능한 웹이다.
  - 검색엔진에 의해 인덱싱 되지 않는다.
- 일반 웹 사이트 도메인과 다른 형태를 보이고 있으며, 철저한 익명화를 특징으로 한다.
- 익명성은 물론 IP 주소 추적이 불가능하도록 고안된 인터넷 영역이다.
- 익명성과 IP 추적을 피할 수 있기에 불법적인 정보 거래에 악용된다.

# Anonymous Network

- 인터넷상에서 개인을 식별하거나 추적하는것을 회피할 수 있도록 해주는 네트워크를 말한다.
- 공격자가 IP 추적이 불가능하도록 자신의 IP를 숨기거나, 속이기 위해 사용하는 대표적인 기술에는 Tor, VPN, Proxy 등이 있다.

## 토르(Tor : The Onion Router)

- 인터넷을 익명으로 검색할 수 있도록 개발된 분산 네트워크 시스템 기반의 서비스이다.
  - 토르 전용 소프트웨어가 필요하다.
- 사용자들이 직접 최종 목적지에 연결하는 대신, 분산된 시스템으로 구성된 릴레이 네트워크(토르 네트워크)를 통해 연결되도록 함으로써, 출발지 IP를 숨실 수 있다.(IP 추적을 어렵게 한다.)

## VPN(Virtual Private Network)

- 가상사설망(VPN)은 공중망(인터넷)을 사설망(전용회선)처럼 이용할 수 있는 기술이다.
- 사용자가 VPN 서버로 암호화된 터널(보안 채널)을 생성한 후, VPN 서버를 통해 목적지에 접속하므로, 출발지 IP를 숨길 수 있다.(IP 추적을 어렵게 한다.)

## 프록시(Proxy)

- 클라이언트와 서버 사이에 프록시 서버(Proxy Server)를 두고, 요청과 응답을 중계해주는 방식으로 서버를 통해 목적지에 접속하므로, 출발지 IP를 숨길 수 있다.(IP 추적을 어렵게 한다.)
- 프록시 서버(Proxy Server)
  - 클라이언트가 자신을 통해서 다른 네트워크 서비스에 간접적으로 접속할 수 있게 해주는 컴퓨터 시스템이다.

# Block Chain

- `분산 데이터 처리 기술로, 네트워크에 참여한 모든 사용자가 거래 내역 등의 데이터를 분산하여 저장하는 기술`이다.
  - 블록들에 정보를 저장하고 체인 형태로 연결한 구조이다.
  - `각 블록은 개인과 개인 간의(P2P) 거래 내역 장부`라고 할 수 있다.
  - 각 블록의 정보는 임의로 수정할 수 없고(위.변조 방지를 위한 암호화 기술 적용) 그 결과는 누구나 열람할 수 있는 특징이 있다.
  - 핵심 암호화 기술 : 해시 알고리즘(함수)
- 블록체인을 `공공 거래 장부` 또는 `분산 거래 장부(원장)`라고도 한다.
- 기존의 중앙 집중형 시스템의 한계를 극복한 `탈중앙화 시스템`을 구현할 수 있다.

# 가상화폐

- 블록체인의 분산 원장 기술을 이용한 디지털 화폐이다.
  - 비트코인, 이더리움 등이 있다.
- 중앙 발행기관 없이 `탈중앙화된 네트워크에서 발행`한다.

## Mining(마이닝)

- 참여자가 가상화폐 블록을 생성하는 과정에 참여하는 것을 마이닝이라 한다.
  - 합의 알고리즘, 작업 증명, 채굴에 따른 보상(암호화폐 지급) 등으로 이뤄져 있다.
    - 합의 알고리즘(consensus algorithm)
      - 다수의 참여자가 하나의 통일된 의사결정을 하기 위해 사용하는 알고리즘이다.
    - 작업 증명(PoW : Proof of Work)
      - 블록을 생성하기 위한 해시 연산 작업을 특정 난이도에 수행하고, 이를 증명한다.
- 마이닝(채굴)을 통해 생성한 `블록 단위로 거래(transaction)` 내역이 기록된다.
